# Spec状態マーキングガイド

> specファイルの冒頭に状態マークを付けて、新しいAIセッションが「これは今も有効か」を即判断できるようにする。

---

## マーク種別

### 廃止（もう使わない）
```markdown
> **廃止（YYYY-MM）**: (廃止理由)。参考として残しています。
```

### 実装済み（specの役割を終えた）
```markdown
> **実装済み（YYYY-MM）**: `src/xxx/yyy.ts` として実装済みです。(補足)
```

### 統合（別specに吸収された）
```markdown
> **廃止（YYYY-MM）**: `specs/xxx/yyy.md` に統合されました。最新はそちらを参照してください。
```

### アーカイブ（時点の記録として保存）
```markdown
> **アーカイブ（YYYY-MM-DD時点の分析）**: 最新は `xxx.md` を参照してください。
```

---

## いつマークするか

- 機能が廃止されたとき
- specの内容が別ファイルに統合されたとき
- 実装が完了してspecが「仕様の記録」に変わったとき
- 古い分析・計画がアーカイブ化されたとき

## ルール

- マークはファイルの**1行目**（タイトルの前）に置く
- ファイル自体は削除しない（過去の意思決定の記録として残す）
- マークには必ず**日付**と**理由**を含める
- 統合先がある場合は**具体的なパス**を記載する
