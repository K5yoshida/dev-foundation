# E2Eテスト生成・実行

> Input: テスト対象の画面またはフロー
> Output: Playwright E2Eテストファイル + 全テスト通過
> 所要時間: 1〜3時間
> フレームワーク: Playwright

---

## 核心: 再帰的クロール＆テスト

```
① 画面のスクリーンショットを撮影
② 全UI要素を列挙（ボタン/リンク/入力/セレクト/トグル/タブ/モーダル）
③ 各要素に対してテストケースを作成・実行
④ 操作の結果、新しい画面状態が発生 → ①に戻る
⑤ 新しい画面状態が発生しなくなるまで繰り返す
```

## 3つの禁止

1. **「到達テスト」禁止**: 遷移できただけで完了にしない。遷移先で②③④を実行
2. **「表面テスト」禁止**: 一部のボタンだけテストして他を無視しない
3. **「浅層偏重」禁止**: 深い階層ほどバグが潜みやすい。深層のテスト密度はL1と同等以上

## テストコードルール

### セレクタ

- `getByRole` > `getByText` > `locator` の優先順位
- strict mode violation 対策: `.first()`, `{ exact: true }`, 領域スコープ

### 待機

- `networkidle` は使わない（APIポーリングで永久に待つ）
- `domcontentloaded` + 特定要素の表示待ちを使う

### 断定性（最重要）

- `.isVisible().catch(() => false)` は禁止
- `if (await element.count() > 0)` は禁止
- 状態分岐が必要な場合は `.or()` パターンのみ

```typescript
const normalState = page.getByText('正常テキスト')
const emptyState = page.getByText('データがありません')
await expect(normalState.or(emptyState)).toBeVisible({ timeout: 15000 })
```

## テストパターン

| 要素種別         | 必須パターン                                           |
| ---------------- | ------------------------------------------------------ |
| ボタン（遷移系） | クリック→URL確認→遷移先内容確認→戻り                   |
| ボタン（削除系） | 確認→キャンセル→データ残存 / 確認→実行→データ消失      |
| テキスト入力     | 空/不正値/最大長超過→エラー、正常値→成功               |
| テーブル         | ソート昇順降順、検索、ページネーション、行クリック     |
| タブ             | 全タブクリック→コンテンツ確認 + タブ内UI要素テスト     |
| モーダル         | 開閉 + 内部の全UI要素テスト + オーバーレイクリック閉じ |

## 実行・修正ループ

全合格するまで自分で修正を繰り返す（最大5ラウンド/ファイル）。
5ラウンドで全合格しない場合はその時点の結果で報告。
