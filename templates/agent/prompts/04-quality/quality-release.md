# リリース前品質チェック

> Input: なし（プロジェクト全体を走査）
> Output: ブロッカー/改善リスト + 修正 + リリース判定
> 所要時間: 2〜4時間
> タイミング: 本番デプロイ前

---

## Phase 1: 自動チェック

```bash
npm run build          # ビルド通過
npm run typecheck      # 型エラー 0件
npm run lint           # Lintエラー 0件
npm run test           # 全テスト通過
npm run widget:size    # 15KB以内
npm run smoke:prod     # 本番smoke通過
```

1つでも失敗したらリリース不可。修正してから再実行。

## Phase 2: 手動コードレビュー

`git log main..HEAD --oneline` で今回リリースに含まれる全コミットを確認。

以下を重点的にチェック:

1. **セキュリティ**: 認証チェック漏れ、RLS未設定、機密情報漏洩
2. **データ整合性**: マイグレーションの破壊的変更、外部キー制約
3. **エッジケース**: 0件、大量データ、同時操作、タイムアウト
4. **ロールバック可能性**: 問題発生時に前のバージョンに戻せるか

## Phase 3: ユーザーフロー検証

主要フローを手動で検証:

1. サインアップ → ダッシュボード到達
2. バナー作成 → プレビュー → 公開
3. CV発生 → ダッシュボード反映
4. パートナー → クライアント管理

## Phase 4: リリース判定

| チェック                            | 結果       |
| ----------------------------------- | ---------- |
| 全自動チェック通過                  |            |
| セキュリティ問題なし                |            |
| データ整合性問題なし                |            |
| 主要フロー検証OK                    |            |
| Business Impact Hypothesis 記載済み |            |
| Ship & Learn Plan 記載済み          |            |
| **リリース判定**                    | GO / NO-GO |

NO-GO の場合: ブロッカーを修正し、Phase 1 から再実行。
