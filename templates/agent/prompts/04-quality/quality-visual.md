# 打鍵テスト（Visual Regression）

> Input: なし（全ページを走査）
> Output: 3デバイスサイズのスクリーンショット + UIバグ修正
> 所要時間: 1〜2時間

---

## Phase 1: スクリーンショット撮影

全ページを3デバイスサイズで撮影:

- Mobile: 375px
- Tablet: 768px
- Desktop: 1440px

ログインが必要なページは必ずログインを行うこと。

## Phase 2: 1ピクセル単位のUIレビュー

全スクリーンショットを以下の観点で精査:

### レイアウト

- 要素の整列（左揃え/中央揃え/右揃えの一貫性）
- 余白・マージンの統一性
- カード高さの不揃い

### レスポンシブ

- ブレークポイント間の急激な変化
- モバイルでの要素の重なり・はみ出し
- フォントサイズの急激な変化

### 視覚的品質

- ヘッダー/フッターの重複・透過問題
- アイコンとテキストの位置ずれ
- 色のコントラスト不足

### コンテンツ

- テキストの切れ・はみ出し
- 画像の欠落
- 空白スペースの過不足

## Phase 3: 問題分類

| 深刻度 | ページ | デバイス | 問題内容 | 修正ファイル |
| ------ | ------ | -------- | -------- | ------------ |

## Phase 4: 修正

優先度の高い順に修正。各修正後に `npm run build` 確認。

## 心構え

- **甘い評価は禁止**: 「問題なし」は疑ってかかる
- **1ピクセルの妥協も許さない**
- **デバイス間の一貫性**: 3デバイス全てで美しく見える
