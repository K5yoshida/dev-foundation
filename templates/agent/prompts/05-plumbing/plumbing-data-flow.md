# データ配管工

> Input: 対象機能またはモジュール名
> Output: データフロー図 + 配管修正（型・バリデーション・エラーハンドリング統一）
> 所要時間: 1〜2時間
> 参照: `docs/11_画面とデータトレーサビリティ.md`, `docs/10_画面とAPIマトリクス.md`

---

## 何をするか

「画面→API→DB」の**縦のデータの流れ**を追跡し、途中で型が変わったり、バリデーションが抜けていたり、エラーハンドリングが不統一な箇所を全て修正する。

水道管の点検と同じ。蛇口（画面）→配管（API）→貯水タンク（DB）を1本ずつ水を通して漏れを見つける。

---

## Phase 1: パイプライン特定

対象モジュールのデータフローを全て列挙する。

```
[画面] ─→ [フォーム入力] ─→ [APIコール] ─→ [バリデーション] ─→ [DB操作] ─→ [レスポンス] ─→ [画面反映]
```

### チェック対象の特定

```bash
# 対象モジュールに関連するファイルを全て見つける
# 例: バナー機能なら
grep -r "banner" app/api/ --include="*.ts" -l
grep -r "banner" components/ --include="*.tsx" -l
grep -r "banner" lib/ --include="*.ts" -l
```

## Phase 2: 6層チェック

各パイプラインを以下の6層で点検する。

| 層                     | チェックポイント                                                  | よくある問題                      |
| ---------------------- | ----------------------------------------------------------------- | --------------------------------- |
| **フロント型**         | フォームの型とAPIリクエスト型が一致するか                         | `string` で送って `number` で期待 |
| **APIバリデーション**  | Zodスキーマで入力を全て検証しているか                             | 必須フィールドのチェック漏れ      |
| **DB型**               | Supabase型（`Database['public']['Tables']`）とAPIの型が一致するか | snake_case ↔ camelCase の変換漏れ |
| **エラーハンドリング** | エラー時にフロントまで適切なメッセージが返るか                    | 500エラーがそのまま表示される     |
| **Null/Undefined**     | 各層で null チェックが適切か                                      | `.data!` で強制アンラップ         |
| **レスポンス型**       | APIレスポンスとフロント側の期待型が一致するか                     | 追加フィールドの見落とし          |

## Phase 3: 型の一筆書き確認

1つのデータフローについて、型が**一筆書きで繋がる**ことを確認。

```typescript
// ✅ 理想: 型が全層で繋がる
type BannerFormData = z.infer<typeof bannerSchema> // フォーム
// ↓ 同じ型でAPI送信
// ↓ bannerSchema.parse(body) でバリデーション
// ↓ supabase.from('banners').insert(data) で保存
// ↓ レスポンスは Database['public']['Tables']['banners']['Row']
// ↓ フロントで Banner 型として受け取り

// ❌ 問題: 途中で型が切れている
// フォームは独自型、APIは別の型、DBはまた別の型
```

## Phase 4: 修正

問題を発見したら以下の優先順で修正:

1. **型の不一致** → 共通型を定義して全層で使う
2. **バリデーション漏れ** → Zodスキーマを追加
3. **エラーハンドリング** → 統一パターンに揃える
4. **Null安全性** → Optional chaining + nullish coalescing

## Phase 5: 検証

```bash
npm run build       # 型エラー 0件
npm run test        # 関連テスト通過
npm run lint        # Lintエラー 0件
```
