# UXフロー最適化

> Input: 対象画面または業務フロー名
> Output: UX改善リスト + 実装
> 所要時間: 1〜2時間
> 参照: `docs/07_業務体験一覧.md`, `docs/23_UI_UXデザインガイドライン.md`, `docs/09_画面一覧.md`

---

## 何をするか

ユーザーが画面を触る**横の流れ**（業務フロー）を追跡し、「迷う・待たされる・やり直す」ポイントを全て潰す。

お客さんが店に入ってから出るまでの導線を歩いてみて、つまずくポイントを直す作業。

---

## Phase 1: フロー特定

対象の業務フローを `docs/07_業務体験一覧.md` から特定し、画面遷移を書き出す。

```
[エントリーポイント] → [画面A] → [操作] → [画面B] → [確認] → [完了]
```

### 主要フロー（参考）

| フロー           | 画面遷移                                                       |
| ---------------- | -------------------------------------------------------------- |
| 初回セットアップ | サインアップ → オンボーディング → ダッシュボード               |
| バナー作成       | ダッシュボード → バナーエディタ → プレビュー → 公開            |
| LP作成           | ダッシュボード → テンプレート選択 → LP編集 → プレビュー → 公開 |
| CV確認           | ダッシュボード → アナリティクス → 詳細                         |
| パートナー管理   | パートナーダッシュ → クライアント一覧 → 代理操作               |

## Phase 2: 5軸レビュー

各画面遷移を以下の5軸でレビューする。

### 1. 認知負荷

- 画面に初めて来たユーザーが3秒で「何をすればいいか」わかるか
- 不要な選択肢が多すぎないか
- プログレスインジケーター（「ステップ2/4」等）があるか

### 2. 操作効率

- 目的達成までのクリック数は最小か
- 入力フォームにデフォルト値・プレースホルダーがあるか
- ショートカット（Ctrl+S で保存等）が使えるか

### 3. フィードバック

- ボタンクリック後にローディング表示があるか
- 成功/失敗がトースト通知で表示されるか
- エラーメッセージが「何をすべきか」を伝えているか

### 4. 復帰性

- ブラウザの戻るボタンで正しく戻れるか
- 途中保存（下書き）ができるか
- 誤操作の取り消し（Undo）が可能か

### 5. エンプティステート

- データが0件のときに適切なガイダンスが表示されるか
- 「最初の〇〇を作成する」ボタンがあるか
- サンプルデータやチュートリアルリンクがあるか

## Phase 3: 問題分類

| 深刻度   | 画面 | 軸  | 問題内容 | 修正方針 |
| -------- | ---- | --- | -------- | -------- |
| Critical |      |     |          |          |
| High     |      |     |          |          |
| Medium   |      |     |          |          |
| Low      |      |     |          |          |

### 深刻度の基準

| レベル   | 基準                               |
| -------- | ---------------------------------- |
| Critical | ユーザーが操作を完了できない       |
| High     | ユーザーが迷う・やり直しが発生する |
| Medium   | 使えるが不便・不安を感じる         |
| Low      | あると嬉しい改善                   |

## Phase 4: 修正

Critical → High → Medium の順に修正。各修正後に `npm run build` 確認。

### 修正時のルール

- UIコンポーネントは `components/ui/` の既存コンポーネントを使う
- Tailwind CSSのユーティリティクラスで実装（インラインスタイル禁止）
- アニメーションは `transition-all duration-200` 程度の控えめなもの
- レスポンシブ: モバイルファーストで実装

## Phase 5: 検証

修正した画面を実際に操作して、5軸の全てで改善されたことを確認。

```bash
npm run build       # ビルド通過
npm run test        # テスト通過
npm run lint        # Lint通過
```

## 完了条件

- [ ] Critical / High 問題が 0件
- [ ] 全フローで「3秒ルール」（3秒で次のアクションがわかる）達成
- [ ] エンプティステートが全画面に実装済み
- [ ] `npm run build` 通過
