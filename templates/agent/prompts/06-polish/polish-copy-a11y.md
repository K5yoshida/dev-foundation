# コピー品質 & アクセシビリティ

> Input: なし（プロダクト全体を走査）
> Output: コピー修正 + アクセシビリティ改善
> 所要時間: 1〜2時間
> 参照: `docs/25_ブランディングとデザインガイド.md`, `docs/23_UI_UXデザインガイドライン.md`

---

## 何をするか

2つの仕事を1回で片付ける:

1. **コピー品質**: 画面上の全テキストを「伝わる日本語」に磨く
2. **アクセシビリティ**: 障害のある方もスクリーンリーダーやキーボードだけで操作できるようにする

---

## Part A: コピー品質チェック

### 走査対象

```bash
# 全コンポーネントのテキストを抽出
grep -rn ">[^<]*<" components/ --include="*.tsx" | head -100
grep -rn "placeholder=" components/ --include="*.tsx"
grep -rn "label=" components/ --include="*.tsx"
grep -rn "toast\." components/ --include="*.tsx"
grep -rn "toast\." app/ --include="*.tsx"
```

### チェック観点

| 観点                         | NG例                   | OK例                                                     |
| ---------------------------- | ---------------------- | -------------------------------------------------------- |
| **動詞で始めるボタン**       | 「送信」「確定」       | 「保存する」「公開する」                                 |
| **具体的なプレースホルダー** | 「入力してください」   | 「例: 株式会社〇〇」                                     |
| **肯定的なエラー**           | 「入力エラー」         | 「メールアドレスの形式で入力してください」               |
| **一貫した敬体**             | 「〜です」「〜だ」混在 | 「〜です」で統一                                         |
| **専門用語の回避**           | 「RLSポリシー」        | 「データアクセス制御」（管理画面でもユーザーが見る箇所） |
| **文字数制限**               | 3行の説明文            | 1行で収まる簡潔さ                                        |

### ブランドトーン

`docs/25_ブランディングとデザインガイド.md` に準拠:

- 親しみやすく、押し付けがましくない
- 専門用語を使わず、中小事業者が直感的に理解できる
- 「簡単さ」「効果」を伝えるコピー

## Part B: アクセシビリティチェック

### 自動チェック

```bash
# eslint-plugin-jsx-a11y が入っていれば
npm run lint  # a11y関連の警告を確認
```

### 手動チェック（7項目）

#### 1. 画像のalt属性

```bash
grep -rn "<img" components/ app/ --include="*.tsx" | grep -v "alt="
grep -rn "<Image" components/ app/ --include="*.tsx" | grep -v "alt="
```

装飾画像は `alt=""`、意味のある画像は内容を説明する alt を設定。

#### 2. フォームラベル

全ての `<input>`, `<select>`, `<textarea>` に `<label>` または `aria-label` があるか。

```bash
grep -rn "<input" components/ --include="*.tsx" -A2 | grep -v "label\|aria-label"
```

#### 3. ボタンのアクセシブル名

アイコンのみのボタンに `aria-label` があるか。

```bash
grep -rn "<Button" components/ --include="*.tsx" -A3 | grep -v "aria-label" | grep "Icon\|icon"
```

#### 4. 色のコントラスト比

- テキストと背景のコントラスト比が 4.5:1 以上（通常テキスト）
- 大きなテキスト（18px以上）は 3:1 以上
- ツールで確認: Chrome DevTools → Lighthouse → Accessibility

#### 5. キーボードナビゲーション

- Tab キーで全てのインタラクティブ要素にフォーカスが当たるか
- Enter / Space でボタン・リンクが動作するか
- Escape でモーダル・ドロップダウンが閉じるか
- フォーカスインジケーター（青い枠線等）が見えるか

#### 6. 見出しの階層

- `h1` → `h2` → `h3` の順序が正しいか（`h1` の次にいきなり `h3` はNG）
- 各ページに `h1` が1つだけあるか

#### 7. ARIA属性

- ローディング中の要素に `aria-busy="true"`
- 展開可能な要素に `aria-expanded`
- エラーメッセージに `role="alert"`

## 修正

コピーとアクセシビリティの問題を発見次第、即修正。

```bash
npm run build
npm run lint
```
