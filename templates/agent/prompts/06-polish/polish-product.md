# プロダクト磨き込み

> Input: なし（プロダクト全体を走査）
> Output: 改善リスト + 実装
> 所要時間: 2〜3時間
> 参照: `docs/23_UI_UXデザインガイドライン.md`, `docs/25_ブランディングとデザインガイド.md`

---

## 何をするか

「動く」は当たり前。「使いたくなる」レベルまで磨き込む。
家で例えると、建物はできた。ここからは内装・照明・家具の配置を詰める段階。

---

## Phase 1: プロダクト全体の棚卸し

全画面をリストアップし、以下の3分類に仕分ける。

| 分類               | 定義                 | 対応                          |
| ------------------ | -------------------- | ----------------------------- |
| **完成**           | 機能もUIも問題なし   | スキップ                      |
| **機能OK・UI未完** | 動くけど見た目が粗い | Phase 2で磨く                 |
| **未実装**         | そもそも画面がない   | 対象外（build-featureで作る） |

## Phase 2: 8つの磨きポイント

「機能OK・UI未完」の画面を以下の8ポイントで磨く。

### 1. ファーストインプレッション（3秒テスト）

画面を開いて3秒で以下がわかるか:

- ここは何の画面か
- 何をすればいいか
- 最も重要な情報はどこか

### 2. マイクロインタラクション

- ボタンのhover/active/disabled状態
- フォーム入力時のフォーカスリング
- トグル/チェックボックスの切り替えアニメーション
- ローディングスケルトン（データ取得中の仮表示）

### 3. エラー状態の美しさ

- エラーメッセージが赤一色でなく、アイコン + テキストで構成
- フォームエラーはフィールド直下に表示
- 全体エラーはトースト通知で表示
- ネットワークエラー時のリトライボタン

### 4. 成功体験の演出

- 保存成功時のトースト通知
- 公開完了時の達成感のある表示
- 初回完了時の褒めるメッセージ

### 5. 一貫性

- ボタンの色使い（プライマリ: 1色のみ）
- 余白のリズム（4px / 8px / 16px / 24px / 32px）
- フォントサイズの階層（見出し / 本文 / 補足 / キャプション）
- アイコンスタイルの統一（Lucide Icons）

### 6. コピーライティング

- ボタンラベルが動詞で始まっているか（「保存する」「公開する」）
- プレースホルダーが具体的か（「例: 株式会社〇〇」）
- 説明文が短く明確か（1文で伝わるか）

### 7. レスポンシブ品質

- モバイル（375px）で全機能が使えるか
- タブレット（768px）でレイアウトが崩れないか
- 横スクロールが発生していないか

### 8. パフォーマンス知覚

- ページ遷移にローディング表示があるか
- 画像にlazy loading が設定されているか
- 大量データ表示にページネーションがあるか

## Phase 3: 優先順位づけ

| 優先度 | 基準                       | 例                                       |
| ------ | -------------------------- | ---------------------------------------- |
| P0     | ユーザーが離脱する         | エラーメッセージなし、レスポンシブ崩壊   |
| P1     | ユーザーが不安になる       | ローディングなし、成功フィードバックなし |
| P2     | プロダクトが安っぽく見える | アニメーションなし、余白不統一           |
| P3     | あると嬉しい               | マイクロインタラクション、達成感演出     |

## Phase 4: 修正

P0 → P1 → P2 の順に修正。P3はリリース後でOK。

```bash
# 各修正後に確認
npm run build
npm run lint
```

## 完了条件

- [ ] P0 / P1 問題が 0件
- [ ] 全画面で3秒テスト合格
- [ ] レスポンシブ（375px / 768px / 1440px）で表示崩れなし
- [ ] `npm run build` 通過
