# AI Governance

## AI生成コードレビューポリシー

### レビュー必須の変更

| カテゴリ            | レビュー要件 | 理由                     |
| ------------------- | ------------ | ------------------------ |
| 認証・認可          | 必須         | セキュリティクリティカル |
| RLSポリシー         | 必須         | テナント間データ漏洩防止 |
| Stripe連携          | 必須         | 決済処理の正確性         |
| ウィジェットJS      | 必須         | 顧客サイトへの影響       |
| DBマイグレーション  | 必須         | データ整合性             |
| API認証ミドルウェア | 必須         | アクセス制御             |

### コードレビューチェックリスト

- [ ] RLSポリシーが正しく設定されているか（テナント分離）
- [ ] 入力バリデーションが行われているか
- [ ] SQLインジェクション対策（パラメータ化クエリ使用）
- [ ] XSS対策（出力エスケープ）
- [ ] 機密情報がログに含まれていないか
- [ ] 環境変数がハードコードされていないか
- [ ] エラーメッセージが内部情報を漏洩していないか
