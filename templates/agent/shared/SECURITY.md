# Security

## セキュリティチェックリスト

### 必須チェック（PR毎）

- [ ] RLSポリシーが正しく設定されているか
- [ ] テナントIDによるデータ分離が担保されているか
- [ ] 入力バリデーションが行われているか
- [ ] 認証チェックが正しく行われているか
- [ ] 環境変数がソースコードに含まれていないか

### 定期チェック（月次）

- [ ] 依存パッケージの脆弱性スキャン（npm audit）
- [ ] Supabase RLSポリシーの監査
- [ ] APIキーのローテーション確認
- [ ] アクセスログの確認

## 秘密情報管理

| 情報             | 保存場所       | ローテーション |
| ---------------- | -------------- | -------------- |
| Supabase Keys    | Vercel環境変数 | 90日           |
| Stripe Keys      | Vercel環境変数 | 90日           |
| SendGrid API Key | Vercel環境変数 | 90日           |
| CloudFlare Token | Vercel環境変数 | 90日           |
| Webhook Secrets  | Vercel環境変数 | 90日           |

## 脆弱性対応フロー

1. `npm audit` で脆弱性検出
2. 重要度評価（Critical/High → 即時対応、Medium → 1週間以内、Low → 次回リリース）
3. パッチ適用 & テスト
4. デプロイ
