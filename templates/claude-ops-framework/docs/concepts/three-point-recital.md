# セッション冒頭 3 点復唱

## 問題

AI はセッション開始時、大量の情報を一度にロードする。その結果、
**最重要原則が埋もれてしまい**、判断の土台がセッションごとに揺らぐ。

例: MEDICA SOERUTE で「間違っていないこと > 充填率」が最上位原則だが、
これを明示的に意識しないと Claude は「効率的な自動化」を提案してしまう。

## 解決策: 3 点復唱を義務化

セッション開始時、AI に**最重要原則を 3 つ復唱**させる。
`SESSION_START.md` にこの手順を明記し、省略禁止とする。

## 3 点の選び方

どのプロジェクトでも以下の 3 種類を選ぶ:

### 1. 過去事故の教訓 (あれば)

**形式**: 「〇〇で××した経験から、△△を避ける」

例:
- MEDICA: 「複数データからマスタを作った時に中身が間違えまくって業務に支障をきたした」
- 金融系: 「為替ヘッジ忘れで想定外損失を出した、以後全ポジションヘッジ必須」

### 2. 最上位原則

**形式**: 「〇〇 > △△」(トレードオフの方向)

例:
- MEDICA: 「間違っていないこと > 充填率」
- スタートアップ: 「速度 > 完璧」
- 医療: 「安全 > 利便性」

### 3. 順序原則

**形式**: 「まず〇〇、△△は後段」

例:
- MEDICA: 「まず SOERUTE を美しく、SF は後段」
- 新規プロダクト: 「まず顧客価値、技術的洗練は後段」
- 組織改革: 「まず小さな成功事例、スケールは後段」

## なぜ 3 点か

- 1 点: 重みが分散せず強力だが、複雑な状況に対応できない
- 2 点: トレードオフ構造を示せるが、順序感が抜ける
- **3 点: 教訓・方向・順序の 3 次元で思考を束縛できる**
- 4 点以上: 復唱が儀式化し、意味が薄れる

## 復唱の形式

AI は SESSION_START.md に従い、以下のように復唱する:

```
3 点復唱:
1. 過去事故: 「複数データからマスタを作った時に中身が間違えまくって業務支障」
2. 最上位原則: 「間違っていないこと > 充填率」
3. 順序原則: 「まず SOERUTE を美しく、SF は後段」
```

一言一句同じで書くのが重要 (記憶への刷り込み効果)。

## 3 点復唱の効果

- **判断の一貫性**: セッションごとに判断軸がブレない
- **事故の予防**: 過去事故の教訓を常時意識
- **優先順位の明確化**: 何を優先すべきかが毎回リセットされる
- **ユーザーへの伝達**: ユーザーも AI の判断基準を共有できる

## アンチパターン

### ❌ 3 点を都度変える

「今日はこの 3 点で」ではなく、**プロジェクト期間中は固定**。
(Phase 切り替え時に改訂はあり得るが、都度変更は避ける)

### ❌ 過去事故がない → 3 点を全て原則にする

過去事故がない場合は 1 点目を空欄にせず、**「想定される失敗シナリオ」**を入れる。
例: 「複数サービス統合時にデータ不整合が出ると、顧客の返金対応で数百万の損失」

### ❌ 抽象的すぎる原則

「ユーザーファースト」「品質重視」のような抽象語は復唱しても効果が薄い。
**「〇〇 > △△」のトレードオフ形式**で具体化する。

## 実装箇所

### SESSION_START.md

```md
## ステップ1: 原則の再確認 (省略禁止)

以下 3 ファイルを必ず読む:
1. past-incident-*.md
2. feedback_principles.md
3. project-ops-master-plan.md

**必ず以下 3 点を復唱してから次へ進む:**
- 過去事故: 「{{PAST_INCIDENT_SUMMARY}}」
- 最上位原則: 「{{TOP_PRINCIPLE}}」
- 順序原則: 「{{ORDER_PRINCIPLE}}」
```

### handoff-template.md Part 1 (プロジェクトオーナーが投げるプロンプト)

```
2. 読了後、3 点復唱:
   - 過去事故: 「...」
   - 最上位原則: 「...」
   - 順序原則: 「...」
```

プロジェクトオーナーが毎セッション冒頭で送るプロンプトに含めることで、
AI が最初のレスポンスで必ず復唱する。

## 長期効果

本フレームワークを実プロジェクトに 2 週間運用した結果 (COF 初出事例):
- AI が原則に反する提案をする頻度が **週 N 回 → ほぼ 0 回** に減少
- プロジェクトオーナーが同じ注意を繰り返す必要がなくなった
- 判断の土台が揺らがないため、Phase 完了判定がスムーズに

3 点復唱は**小さな仕組みだが効果が大きい**。
