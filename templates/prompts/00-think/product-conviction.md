# プロダクト確信度チェック

> Input: 検討中の機能名
> Output: GO / PIVOT / KILL 判定 + CONVICTION_LOG 記録
> 所要時間: 30分〜1時間
> タイミング: spec-create の前、新機能の着手前、戦略変更時

---

## WHY

HOW（プロダクト）をいくら精密に作っても、WHO（誰に）× WHAT（どんな価値を）が間違っていれば全てゴミ。
このプロンプトは、実装に入る前に「作るべきか？」を問う最上位のチェックポイント。

```
WHO（誰に）× WHAT（どんな価値を）= 売れるか？
       ↓
市場規模 × 競合性 × モート = 維持・拡大できるか？
       ↓
HOW = プロダクト ← 01〜08のプロンプトが扱う範囲
```

---

## Phase 1: ビジネス構造の確認

CLAUDE.md の「設計書クイックリファレンス」（または同等のセクション）から、ビジネス戦略に関するドキュメントを特定して読む。

抽出する情報:
- **WHO**: ターゲット顧客のペルソナと課題
- **WHAT**: 提供する価値提案（ベネフィット）
- **市場**: 市場規模、成長性
- **競合**: 主要競合とポジショニング
- **モート**: 防御壁（参入障壁）
- **成長レバー**: どのように拡大するか

## Phase 2: 市場の現実チェック

### AI実行可能
- 競合プロダクトの直近アップデートをWeb検索
- 設計書の競合分析と現実のギャップを特定
- 類似サービスの価格帯・機能比較

### 人間アクション候補（出力のみ）
- ターゲット顧客3人にヒアリング
- 解約理由・離脱理由の確認
- パートナー・チャネルからの声

## Phase 3: 5つの根本質問

| # | 質問 | 確信度(1-5) |
|---|------|------------|
| Q1 | **WHO接続**: この機能は、設計書に定義されたターゲットのどの課題を解決するか？ | |
| Q2 | **WHAT接続**: この機能は、設計書に定義された価値提案のどれを実現するか？ | |
| Q3 | **モート**: 競合がこれを真似するのに何ヶ月かかるか？ | |
| Q4 | **必須性**: この機能がなくても設計書の目標を達成できるか？ | |
| Q5 | **Ship翌日テスト**: 明日リリースしたら、ターゲット顧客に「使ってください」と言えるか？ | |

### 判定

| 合計スコア | 判定 | アクション |
|-----------|------|-----------|
| 20-25 | **GO** | spec-create に進む |
| 15-19 | **PIVOT** | 機能のスコープを変更して再評価 |
| 5-14 | **KILL** | この機能を中止。別の機能を検討 |

## Phase 4: 優先順位の再評価

GOの場合でも:
- 今作るべきか？（他にもっとインパクトの大きい機能は？）
- 最小限のスコープは？（フルスペックでなく、検証に必要な最小単位は？）

## Phase 5: CONVICTION_LOG 記録

`.claude/CONVICTION_LOG.md` に以下を記録:
- 日付、機能名、判定（GO/PIVOT/KILL）
- 各質問のスコアと根拠
- PIVOTの場合: 変更内容
- KILLの場合: 代替案

## 完了条件

- [ ] Phase 1 でビジネス戦略ドキュメントを全て読んだ
- [ ] Phase 2 で市場の現実チェック実施
- [ ] Phase 3 で5つの質問に全て回答
- [ ] 判定が GO / PIVOT / KILL のいずれかに確定
- [ ] CONVICTION_LOG に記録済み
