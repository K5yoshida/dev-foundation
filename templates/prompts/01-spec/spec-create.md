# P0スペック作成

> Input: 機能名
> Output: specs/{{機能名}}/spec.md + plan.md + tasks.md
> 所要時間: 2〜3時間

---

## 何をするか

設計書から1つの機能を取り出し、実装に必要な全情報を1つのスペックに凝縮する。
料理で例えると、レシピ本（設計書）から今日作る料理のレシピだけを抜き出して、材料・手順・完成写真を1枚にまとめる作業。

## 手順

### Step 1: 設計書リーディング

対象機能に関連する設計書を全て読む:
- 機能一覧 → 何を作るか
- 画面一覧 → どう見えるか
- DB設計 → データをどう持つか
- API仕様 → どう通信するか

### Step 2: spec.md 作成

```markdown
# {{機能名}} スペック

## 概要
何をする機能か、1〜2文で。

## ユーザーストーリー
「〇〇として、△△したい。なぜなら□□だから。」

## 画面仕様
- 画面名、URL、レイアウト
- 各UI要素の挙動

## データ設計
- テーブル構造
- リレーション

## API仕様
- エンドポイント、メソッド、リクエスト/レスポンス

## エッジケース
- 0件、大量データ、同時操作、タイムアウト

## セキュリティ
- 認証/認可、入力バリデーション、RLS
```

### Step 3: plan.md 作成（実装計画）

フェーズ分けして実装順序を決める。

### Step 4: tasks.md 作成（タスクリスト）

チェック可能な形式で全タスクを列挙。

## 完了条件

- [ ] spec.md に曖昧な表現が0
- [ ] エッジケースが5件以上列挙
- [ ] セキュリティチェック項目あり
