# 実装計画作成

> Input: 承認済み spec.md
> Output: plan.md + tasks.md
> 所要時間: 1〜2時間

---

## 手順

1. spec.md を読み込む
2. 実装をフェーズ分割（DB → API → フロント → テスト）
3. 各フェーズのタスクを粒度を揃えて列挙
4. 依存関係を整理（並列実行可能なものを特定）

## plan.md テンプレート

```markdown
# {{機能名}} 実装計画

## フェーズ1: DB層
- マイグレーション作成
- RLSポリシー追加

## フェーズ2: API層
- エンドポイント実装
- バリデーション追加

## フェーズ3: フロント層
- コンポーネント実装
- 画面結合

## フェーズ4: テスト
- ユニットテスト
- E2Eテスト
```

## 完了条件
- [ ] 全タスクが1〜2時間以内の粒度
- [ ] 依存関係が明確
