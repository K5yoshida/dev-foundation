# データ配管工

> Input: 対象機能またはモジュール名
> Output: データフロー図 + 配管修正
> 所要時間: 1〜2時間

---

## 何をするか

画面→API→DBの縦のデータの流れを追跡し、型の不一致・バリデーション漏れ・エラーハンドリングの不統一を修正する。

## 6層チェック

| 層 | チェックポイント |
|----|----------------|
| フロント型 | フォームの型とAPIリクエスト型が一致するか |
| APIバリデーション | Zodスキーマで入力を全て検証しているか |
| DB型 | DB型とAPIの型が一致するか |
| エラーハンドリング | エラー時にフロントまで適切なメッセージが返るか |
| Null/Undefined | 各層でnullチェックが適切か |
| レスポンス型 | APIレスポンスとフロント側の期待型が一致するか |

## 完了条件
- [ ] 全パイプラインの型が一筆書きで繋がる
- [ ] 全APIルートにZodバリデーションあり
- [ ] `npm run build` 通過
